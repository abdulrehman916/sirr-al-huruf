# SIRR AL-HURUF — RESEARCH LAW
## Permanent Project-Wide Policy · The 28 Birhatīya (Barhatiah) Cards

**Established:** 2026-07-19 (Asia/Dubai)
**Scope:** Every future update of Section C (`HolyNameEsotericKnowledge`, HNK-MHC-001 … HNK-MHC-028) and every function, page, or component that reads or writes it (`ingestSectionCManuscript`, `enrichSectionCScholarly`, `auditSectionCQuality`, `HolyNameEsotericResearchProfile`, `SectionCNames`).
**Status:** PERMANENT and IMMUTABLE. This law applies to every future update of Section C, in every version, forever. It overrides any earlier "complete / final / frozen" statement. It works together with — and never relaxes — `SECTION_C_LIVING_LIBRARY_LAW.md`.

---

## The 18 Laws

**1. Every card is permanently expandable.**
A card is a living record, never a finished one. New fields, new sections, and new entries may be added at any time.

**2. No card is ever considered final.**
"Version 1" is a baseline, not a destination. Version 2, 3, … follow indefinitely. There is no terminal version.

**3. New authentic information may always be appended.**
Whenever an authentic book, manuscript, archive, or reliable scholarly website documents new content for a name, it is appended to the matching card — no permission is needed beyond the source being authentic.

**4. Existing information must never be deleted.**
No record, entry, field, source, or letter is ever removed — not by ingestion, not by enrichment, not by audit, not by "cleanup." Append-only, forever.

**5. Existing information must never be rewritten.**
Original Arabic, transliterations, meanings, Abjad values, and citations are preserved verbatim. Corrections and alternates go into the dedicated `alternate_*` arrays as *new* entries; the original is kept alongside them.

**6. Every source remains independent.**
Book A and Book B are never combined. Each source's content is stored as its own entry with its own full citation, even when two sources describe the same method.

**7. Conflicting scholarly opinions must always remain separate.**
Where scholars disagree ("some say… others say…"), every opinion is kept as its own attributed entry. They are never merged, averaged, or summarized into one statement.

**8. Every statement must cite its source.**
No uncited statement enters a card. Each entry carries book name · author · edition · publisher · page number · URL (when from the internet) · confidence level. A statement without a citable source is not appended.

**9. Every quotation must preserve the original Arabic exactly.**
Arabic is copied verbatim — every letter, harakah, hamza, madd, shaddah, sukoon, tanween, and punctuation exactly as printed. Never normalized, never transliterated-then-reconstructed, never "improved."

**10. Malayalam explanations must always be natural, fluent, and easy to understand.**
Malayalam is written as a Malayalam scholar would write it — grammatical, fluent, readable — never machine-translation style. Arabic words embedded in the source are kept in Arabic verbatim within the Malayalam text.

**11. Unknown information must never be invented.**
If a source does not document a field, the field stays empty and displays the Malayalam placeholder. Empty + honest is always preferred over filled + guessed.

**12. AI must never fabricate rituals, awfāq, talismans, khawāṣṣ, or mujarrabāt.**
No ritual, magic square, talisman, khawāṣṣ, mujarrab, duʿā, or invocation is generated by AI. They come only from verbatim transcription of an uploaded source, or from an explicit, owner-authorized internet-research pass that cites a real source. AI may translate, classify, and attribute — never invent ritual content.

**13. If authenticity cannot be verified, clearly mark the information as "Unverified."**
Unverifiable entries are stored with `confidence: LOW` / `verification_status: needs_review` and shown with an explicit "Unverified" badge. They are never presented as established fact until a verifying source is added.

**14. Every future PDF uploaded by the user must automatically merge into existing cards without destroying previous knowledge.**
PDF ingestion (`ingestSectionCManuscript`) appends to existing cards. It matches content to the existing 28 names, deduplicates only by exact `(text + source + page)`, and never overwrites or deletes prior entries. General whole-conjuration content goes to card #1's shared sections; name-specific content goes to its matching card.

**15. Every future internet research must enrich the existing cards instead of replacing them.**
Internet research (`enrichSectionCScholarly`) runs only on explicit owner authorization. It appends new findings to the existing card's arrays, records the URL and confidence level on each entry, and never replaces primary fields. Alternate spellings, pronunciations, meanings, and Abjad values go to the `alternate_*` arrays.

**16. Keep complete version history for every card so additions from future sources are traceable.**
Each append records its source, timestamp, and confidence. The `sources[]` list and every advanced-section entry carry `source_reference` + `source_page` (+ `url`). The audit function (`auditSectionCQuality`) can re-verify integrity at any time without deleting anything. Old entries are never edited out of history.

**17. Every image, magic square, awfāq, or manuscript page must keep its original source reference.**
Every image URL stored on a card carries its originating source (book, page, manuscript, or screenshot). Images are original scans only — never redrawn or AI-recreated. A talisman image without a source citation is not appended.

**18. Every card should eventually become the definitive scholarly reference for that Birhatiya name.**
The long-term goal of each card is to be the world's most comprehensive, fully-cited, multi-source, verbatim-accurate reference for that single name — gathering, over time, every authentic duʿā, invocation, mujarrab, khawāṣṣ, amal, awfāq, magic square, talisman, khādim, condition, repetition, timing, day, planetary hour (only when sourced), incense, method, benefit, warning, scholarly opinion, historical note, manuscript variant, and image, each independently attributed.

## Enforcement

- This law is enforced by architecture, not by trust: the ingestion and enrichment functions use append-only `.push` with `(text + source + page)` dedup; the schema provides dedicated `alternate_*` arrays for variants; the UI renders each `scholarly_data` and advanced-section entry as an independent item with its own citation.
- Any future change to Section C code that would violate one of these 18 laws must be rejected. When unsure, STOP and ask the owner for confirmation.
- This law does not relax `SECTION_C_LIVING_LIBRARY_LAW.md`; both stand together. Where they overlap, the stricter rule applies.

## Acceptable Sources (priority)

1. Classical manuscripts and authenticated critical editions (e.g. Aḥmad al-Būnī, *Manbaʿ ʾUṣūl al-Ḥikma*).
2. Printed scholarly books with full bibliographic data.
3. Academic archives and digitized manuscript libraries (archive.org, etc.).
4. Reliable scholarly websites — only on explicit owner authorization, with a working URL + confidence level.

Fabricated, AI-guessed, or unsourced content is never appended.
